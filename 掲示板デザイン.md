## 1. プロジェクト概要
日本の「2ちゃんねる」に代表される、クラシックなUI/UXを持つテキストベースの匿名掲示板サイトを、**全国の地域ごとに独立した掲示板を持つポータルサイト**として構築する。

ユーザーはトップページから地域を選択し、その地域専用の掲示板（スレッド一覧）にアクセスできる。各地域の掲示板では、スレッドを作成し、その中で匿名でコミュニケーションを行うことができる。主な機能はスレッド作成、レス投稿、sage/age機能、トリップ機能、投稿削除機能、そして地域別掲示板管理機能である。

## 3. デザイン＆レイアウト仕様

### 全体レイアウト:
- **構成**: 固定幅（最大1000px）の中央揃えレイアウト。
- **背景色**: ページ全体の背景はダークグレー (#333333)、メインコンテンツエリアの背景は薄いクリーム色 (#f0f0e0) とする。
- **カラム**: メインコンテンツとサイドバーの2カラム構成。幅の比率はメインが3、サイドバーが1。カラム間の余白は20px。

### タイポグラフィ:
- **フォント**: 日本語環境を考慮し、"MS PGothic", "Hiragino Kaku Gothic ProN", "メイリオ", Meiryo, sans-serif の順でフォントを指定する。
- **カラー**: 基本テキストは黒 (#000000)、リンクは青 (#0000EE)、投稿者名は緑 (#008000) とする。

### コンポーネントスタイル:
- **ボタン**: 主要な操作ボタン（投稿ボタンなど）は、背景色をマゼンタ (#e5007f)、文字色を白 (#ffffff) とし、視覚的なアクセントとする。
- **フォーム**: labelとinput/textareaが横に並ぶレイアウト。labelの幅は固定し、入力欄は可変幅とする。
- **投稿区切り**: 各投稿（レス）は、下線 (border-bottom) で視覚的に区切る。
- **地域選択ボタン**: 背景色を落ち着いた緑 (#6b8e23)、文字色を白 (#ffffff) とし、角丸の長方形で表示。2列グリッドレイアウトで配置。

## 4. データベーススキーマ (schema.prisma)
以下のスキーマを定義する。PostモデルのdeletePasswordは必ずハッシュ化して保存すること。

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model Region {
  id    Int    @id @default(autoincrement())
  name  String @unique @db.VarChar(50) // 例: 「北海道」「東北」「関東」
  slug  String @unique @db.VarChar(50) // URL用スラッグ 例: "hokkaido", "tohoku"
  order Int    // 表示順序
  areas Area[]
}

model Area {
  id       Int      @id @default(autoincrement())
  name     String   @db.VarChar(50)   // 例: 「札幌」「函館」「東京」「大阪」
  slug     String   @unique @db.VarChar(50) // URL用スラッグ 例: "sapporo", "tokyo"
  regionId Int
  region   Region   @relation(fields: [regionId], references: [id])
  threads  Thread[]
}

model Thread {
  id        Int      @id @default(autoincrement())
  areaId    Int      // どのエリアに属するスレッドか
  area      Area     @relation(fields: [areaId], references: [id]) // 関連付けを追加
  title     String   @db.VarChar(255)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt // レス投稿時に更新（sageの場合は更新しない）
  posts     Post[]
}

model Post {
  id              Int      @id @default(autoincrement())
  threadId        Int
  thread          Thread   @relation(fields: [threadId], references: [id], onDelete: Cascade)
  postNumber      Int      // スレッド内での投稿番号 (1から始まる)

  name            String?  @db.VarChar(100)
  email           String?  @db.VarChar(100) // "sage"判定に使用
  body            String   @db.Text

  trip            String?  @db.VarChar(50)  // 生成されたトリップ
  userId          String   @db.VarChar(50)  // 日付とIPから生成されるID

  deletePassword  String?  @db.VarChar(255) // ハッシュ化された削除パスワード
  ipAddress       String   @db.VarChar(50)  // サーバー側で記録

  createdAt       DateTime @default(now())

  @@unique([threadId, postNumber]) // 同じスレッドで投稿番号が重複しないようにする
}
```

**データ投入の前提**: RegionとAreaのデータは、開発者が事前に手動で投入（シーディング）しておく必要があります。（例: 北海道地方に札幌、函館...、関東地方に東京、神奈川...など）

## 5. 機能要件

### 5.1. ページ構成
- **トップページ (/)**: 
  - 役割: 地域選択のポータルページ
  - 表示要素:
    - サイト全体の紹介文
    - 注目ユーザーのアイコン一覧（プロモーション要素）
    - 地域ブロック（例: 北海道、東北、関東）がタイル状に並ぶ
    - 各地域ブロック内には、所属するエリア（例: 札幌, 函館）へのリンクをリスト表示

- **エリア（地域）別トップページ (/areas/[area_slug])**:
  - 役割: 特定のエリア（例: 札幌）に特化した掲示板のトップページ
  - URL例: /areas/sapporo, /areas/tokyo
  - 表示要素:
    - そのエリアのスレッド一覧
    - そのエリアで新しいスレッドを作成するフォーム
    - サイドバーには、同じ地方（例: 北海道）の他のエリアへのリンクを表示

- **スレッドページ (/threads/[id])**: 
  - 役割: 特定スレッドの投稿一覧とレス投稿フォームを表示
  - 表示要素:
    - パンくずリストを追加し、ユーザーが現在地を把握しやすくする
    - （例: トップ > 北海道 > 札幌 > スレッドタイトル）

### 5.2. コア機能と特殊ロジック

#### 地域選択:
- ユーザーはトップページから、まず大きな地方（関東など）を選び、次により詳細なエリア（東京など）を選択して、目的の掲示板に移動する。

#### スレッド作成:
- スレッド作成フォームは、必ず特定のエリアページ (/areas/[area_slug]) に設置する。
- 作成されたスレッドは、そのページのエリアに自動的に紐付けられる (areaId が設定される)。
- タイトルと最初の投稿内容を入力して新しいスレッドを作成する。

#### レス投稿:
- スレッドページから名前、E-mail、本文、削除PASSを入力して投稿する。

#### sage/age機能:
- **age (デフォルト)**: E-mail欄がsageでない投稿の場合、親スレッドのupdatedAtを更新し、スレッド一覧で最上部に表示させる。
- **sage**: E-mail欄にsage（半角/全角不問）と入力された投稿の場合、親スレッドのupdatedAtを更新しない。

#### トリップ機能:
- 名前欄に `名前#トリップキー` の形式で入力された場合、#以降をキーとしてトリップを生成する。
- 生成されたトリップは `◆XXXXXXXXXX` の形式で名前の後に表示する。
- 生成ロジックは、2ch互換のアルゴリズム（ソルトを用いたcryptまたはそれに準ずるハッシュ化）を実装すること。

#### ID表示機能:
- 投稿者のIPアドレスと投稿日付から一意の文字列（例: ID:AbCdEfG1）を生成し、userIdとして表示する。
- 同じユーザーでも日付が変わればIDも変わる。

#### 投稿削除機能:
- 各投稿に削除リンクを設置。
- リンククリック後、パスワード入力を求め、投稿時に設定したパスワードと一致すれば投稿を削除する（本文を「削除されました」に更新する論理削除を推奨）。

#### ナビゲーション:
- ヘッダーやサイドバーに、常に他のエリアやトップページへ戻るためのリンクを配置する。

## 6. UIコンポーネントごとの詳細

### 地域選択コンポーネント (トップページ):
- 大きな角丸の長方形ボタンで各地方（北海道、東北など）を表示。
- ボタンはグリッドレイアウトで2列に配置。
- 各ボタン内には、所属するエリア名（札幌, 仙台...）を小さな文字でリスト表示する。
- RegionとAreaのデータから動的に生成する。
- スタイルは、背景色を落ち着いた緑 (#6b8e23のような色味)、文字色を白とする。

### スレッド一覧 (エリア別トップページ):
- テーブル形式で、番号, スレッドタイトル, レス数 を表示。
- タイトルはスレッドページへのリンクとする。
- リストはスレッドのupdatedAt（更新日時）が新しい順にソートする。

### レス一覧 (スレッドページ):
- 各レスのヘッダーに 投稿番号, 名前, E-mail, 投稿日時, ID, トリップ を表示。
- 投稿本文はpreタグや改行の自動変換を適用し、整形して表示する。

### 投稿フォーム (共通):
- フィールド: 名前, E-mail, 本文(必須), 削除用PASS。
- 本文の未入力を禁止するクライアント/サーバー両面でのバリデーションを実装する。

### パンくずリストコンポーネント (Breadcrumbs):
- スレッドページなどで表示。
- トップ > {地方名} > {エリア名} > {スレッドタイトル} のように階層構造を示す。
- 各階層はリンクになっており、上位のページに戻れるようにする。

## 7. APIエンドポイント (Route Handlers)
以下のAPIを `/app/api/` 以下に実装すること。

- **GET /api/regions**: 全ての地方と、それに属するエリアの一覧を取得する（トップページ表示用）。
- **GET /api/threads?area=[area_slug]**: 変更。areaクエリパラメータで指定されたエリアのスレッド一覧を取得するようにする。
- **POST /api/threads**: 変更。リクエストボディに areaId を含め、どのエリアのスレッドかを指定して作成するようにする。
- **GET /api/threads/[id]**: 特定スレッドの投稿一覧を取得（ページネーション対応）。
- **POST /api/threads/[id]/posts**: 特定スレッドにレスを投稿。
- **DELETE /api/posts/[id]**: 投稿を削除（リクエストボディでパスワードを受け取り認証）。変更なし。

## 8. エラーハンドリングとセキュリティ
- 存在しないスレッドへのアクセス時は404ページを表示する。
- APIエラー発生時は、ユーザーに適切なフィードバックメッセージを表示する。
- XSS（クロスサイトスクリプティング）対策として、ユーザーからの入力は適切にエスケープ処理を行うこと。

### 主な実装のポイント

#### トリップ機能の実装
- 名前欄の「#」以降をキーとしてSHA-1ハッシュを生成
- 生成されたハッシュをBase64エンコードして10文字に切り詰め
- 「◆」プレフィックスを付けて表示

#### sage/age機能の実装
- E-mail欄の入力値が「sage」（大文字小文字・全角半角不問）の場合
- スレッドのupdatedAtを更新しない
- それ以外の場合は、投稿時にスレッドのupdatedAtを現在時刻に更新

#### ユーザーID生成
- IPアドレスと日付（YYYY-MM-DD形式）を組み合わせてシード値を作成
- SHA-256でハッシュ化し、8文字のIDを生成
- 「ID:」プレフィックスを付けて表示

#### 投稿削除機能
- 削除パスワードはbcryptでハッシュ化して保存
- 削除時はパスワードを検証し、一致した場合のみ論理削除
- 削除された投稿は本文を「あぼーん」に置換
